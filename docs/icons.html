Icon Workflow

1. Icon Creation

Each icon is first created in a Vector format to maintain the same native rendering that SVG uses. Icons are carefully crafted to reduce points and file size. Icons are placed on a grid and the icon artboard grid to keep sizing and resolution consistent.

2. Administrate Library

Icons are exported from Illustrator into an individual SVG file for each icon. The icons are then imported into the Design System library hosted at IcoMoon.io.

3. Download Icon Library

The icon library is edited and maintained as needed within the IcoMoon app. Next, IcoMoon then generates and exports all of the font files and CSS needed to use the font.

4. Update Design System

When the icon library is up-to-date at IcoMoon, the final library is exported to font files and cascading style sheets and later imported it into the Design System repository.

How Icons Work

Webfont icons work by using CSS to inject a specific icon into the HTML using the CSS content property. It then uses @font-face to load a the referenced icon from the webfont typeface. First, the webfont is refereneced withing a style sheet and then imported and flattened into the overall design system's CSS file.

Icon Sizing

Because the icons are used within a font, all icons should be sized by using the CSS font-size property. It is important to use or create a parent class around the icon span to ensure the font-size for only that particular icon. Without this parent class, every icon of that type in the entire application will receive the font-size change. It's recommended that a each section have a semantic class name for these scenarios.

Icon Color

Because the icons are used within a font, all icons should be sized by using the CSS color property. It is important to use or create a parent class around the icon span to ensure the color for only that particular icon. Without this parent class, every icon of that type in the entire application will receive the color change. It's recommended that a each section have a semantic class name for these scenarios.

Selecting Icons

In most cases a designers should be supplying or indicating which icon to use in different scenarios or states. However, we do realize that there are times when another engineer or designer may need to select an icon. In these cases, it's important to be select an icon that makes sense for the view or state. Here are a few rules to keep in mind when selecting an icon:

  • Do not use an icon that is globally used (for example the down triangle is used for dropdown menus)
  • Do not pick an icon as a placeholder
  • Does the icon make sense contextually within the interface?
  • Is an icon needed?

Request an Icon

There are many cases where the right icon isn't available in the library. The UX team is glad to create one for you! Please send us an email to request and icon. Please be sure to provide some screenshots and a brief summary. ux-team@ascension.org

Usage

Using icons and webfonts is very easy using Schema. The only dependency is loading Schema Design System into your web application's HTML Head. This is a standardized step to use all of Schema, so nothing else is needed. Simply apply the icon's CSS class to an HTML span and the icon will appear. It can be sized using the CSS font-size property.
